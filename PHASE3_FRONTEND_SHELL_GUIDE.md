# Phase 3 Frontend Shell

This frontend shell is completely independent from the .NET projects and can be managed in a separate repository.

## What is included
- Vite + React + TypeScript setup
- Mantine UI shell
- Mock channel selector, question table, test runner, results table, and history selector
- No backend API dependency is required to view the shell

## Folder structure

## How to test before generating the final JS package
1. Install Node.js 18 or later.
2. Open the `phase3-frontend-shell` folder.
3. Run `npm install`.
4. Start the dev server with `npm run dev`.
5. Open the local URL shown by Vite.
6. Verify the shell renders the header, channel selector, question table, history controls, and results panel.
7. Click **Run selected** and **Run all** to confirm the mock sequential update behavior.
8. Click **Reset mock data** to restore the initial state.
9. After UI verification, run `npm run build` to generate the single-file production bundle.

## Expected behavior in shell mode
- Channel selection updates the visible question bank.
- Run actions simulate sequential execution with mock scores.
- History controls are UI-only placeholders until backend APIs are wired later.
- No .NET project references, shared libraries, or solution files are required.

## Build output
The production build will be generated by Vite and can later be packaged as a single JS file using the configured single-file plugin.
